# Sales Dashboard Case Study

## Introduction
This project analyzes a retail sales dataset to identify revenue drivers, top customers, and sales trends.  
The goal was to build a recruiter‑friendly Excel dashboard showcasing key KPIs.

## Problem Statement
Businesses often struggle to quickly identify which countries, products, and customers generate the most revenue.  
This project demonstrates how structured data analysis can solve that problem by combining technical skills with clear communication.

## Workflow
The project followed a structured workflow:
- **Data Cleaning** → handled duplicates, missing values, formatted dates.
- **Pivot Tables** → built KPIs for revenue, products, customers, and trends.
- **Dashboard** → added Pivot Charts, slicers, and insights summary.

## Key KPIs
- **Revenue by Country** → Identified top markets.
- **Revenue by Product** → Highlighted best‑selling items.
- **Top 10 Customers** → Listed most valuable clients.
- **Monthly/Quarterly Trends** → Showed seasonality and growth.

## Insights
- **Revenue by Country** → United Kingdom alone contributed 81% of total revenue.  
- **Revenue by Product** → Regency Cakestand (3‑tier) is the top‑selling product.  
- **Top 10 Customers** → The top 10 customers contributed revenue of ₹1,368,640.64, which is 16.5% of total revenue.  
   - Customer ID 14646 alone contributed around ₹279,489.02, which is 3.3% of total revenue.  
- **Monthly/Quarterly Trends** → Sales peaked in November 2011, showing strong seasonality in Q4 of 2011.  

## Conclusion
This project demonstrates end‑to‑end data analysis skills applied to a real sales dataset.  
From cleaning and preparing the data to defining KPIs and building dashboards.
By building Pivot Tables, Pivot Charts, and slicers, I created an interactive dashboard that highlights revenue drivers, top customers, and seasonal trends.  
It mirrors real analyst tasks — cleaning data, structuring workflows, and presenting recruiter‑friendly insights.
